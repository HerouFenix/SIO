        2.2.2
                "Look for an action that stores a message into the server. For it to be a Server Side Attack, the payload must be included in the web page when the page is built by the server."
                
                        Se formos a um post e comentarmos, podemos reparar que o campo do nome está protegido contra tags MAS, a mensagem aceita tags novas (tipo <img> ou <script>) !!
                                Exemplo: Meter na mensagem: <script>alert("LMAO GOTTEM")</script> , conseguimos inserir novo código no DOM

                                Isto é Server Sided Store Cross Side Scriptiing (Stored XSS) !
